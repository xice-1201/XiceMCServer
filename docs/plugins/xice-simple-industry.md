# XiceSimpleIndustry

`XiceSimpleIndustry` 是 XiceMCServer 的简易工业插件，用于提供插件侧模拟的自定义机器方块。

## 简易刷石机

首个机器为“简易刷石机”。它使用插件数据保存机器状态，实际承载方块为不可见的屏障方块，外观由资源包中的 `xicesimpleindustry:simple_cobblestone_generator` 模型通过展示实体显示，因此不会覆盖或改变任何原版方块材质。

功能：

1. 拥有 `give` 动作授权的玩家可通过 `/xicecustomitem give <物品ID> [玩家] [数量]` 发放简易刷石机、亡灵粉尘或亡灵核心；该授权由 Web 权限页写入 `XiceCustomItem` 运行时配置。
2. 玩家放置机器时，机器正面会朝向玩家，支持朝上、朝下和四个水平方向。
3. 玩家右键机器打开 UI。
4. UI 中只有输入槽允许放入水桶、熔岩桶和空桶。
5. 点击确认时，水桶会变为空桶并让水量 +1，上限 16；熔岩桶会变为空桶并让熔岩 +1，上限 64；空桶不会触发变化。
6. 效率值等于 `min(熔岩值, 水量 * 4)`。
7. 机器未被红石充能时，每 30 tick 向正面方向吐出一组圆石，数量等于当前效率值。
8. 机器被红石充能时暂停吐出圆石。
9. 指向机器的漏斗可以向输入槽输入水桶、熔岩桶或空桶；机器下方的漏斗可以从输入槽抽取这些物品。
10. 机器接收到红石脉冲时，会执行一次“确认注入”。
11. 玩家潜行右键机器时不会打开 UI。
12. 成功注入水或熔岩时，会在机器位置播放对应的倾倒液体音效。
13. 如果机器正面被容器阻挡，产出的圆石会优先进入该容器库存；容器放不下的剩余圆石才会喷出。
14. 生存玩家破坏机器时，会按投掷器硬度模拟正常挖掘速度；镐子材质和效率附魔会影响挖掘进度。

## 合成

```text
玻璃  TNT    玻璃
空桶  黑曜石 空桶
铁锭  红石粉 铁锭
```

配方中的玻璃可使用普通玻璃、任意染色玻璃或遮光玻璃。

玩家获取红石粉后，会自动解锁简易刷石机的配方书配方；如果玩家上线时背包中已有红石粉，也会补发该配方解锁。

## 简易村民繁殖机

“简易村民繁殖机”是用于缓慢产出村民刷怪蛋的自定义机器方块。玩家右键机器可打开 UI：

1. 输出槽只允许取出村民刷怪蛋，不能放入任何物品。
2. 开关默认关闭，玩家点击开关可切换开机/关机。
3. 状态指示标会显示当前开关状态和计时进度。
4. 内置计时器为 5 分钟。关机时计时仍会继续，最多累积到计时上限；下一次开机时若计时已满，会立刻尝试输出 1 个村民刷怪蛋。
5. 开机状态下每次计时完成会尝试向输出槽加入 1 个村民刷怪蛋；如果输出槽已满，本轮不输出，但计时器仍会重新开始。

配方：

```text
床             床     床
村民刷怪蛋     胡萝卜 村民刷怪蛋
泥土           水桶   泥土
```

配方中的床可使用任意颜色床；胡萝卜可替换为马铃薯、甜菜根或小麦。合成后水桶按原版容器物品规则变为空桶并保留在合成栏。玩家获取过村民刷怪蛋后，会自动解锁简易村民繁殖机、简易村民交易站和僵尸村民刷怪蛋的配方书配方。

僵尸村民刷怪蛋无序配方：

```text
村民刷怪蛋 + 亡灵粉尘
```

非创造模式玩家无法使用任何刷怪蛋与刷怪笼交互，避免误改原版刷怪笼类型。

## 简易村民交易站

“简易村民交易站”用于把刷村民交易的流程固化到一个可搬运的机器方块中。玩家右键未设置职业的交易站会打开求职 UI：

1. 输入槽只允许放入 1 个原版村民工作方块。
2. 点击确认后，交易站会记录该工作方块对应的村民职业，并进入交易配置 UI。
3. 职业、已选交易项会随机器保存；机器被挖掘后再放置不会丢失这些配置。

决定职业后，玩家右键交易站会打开交易配置 UI。UI 中可以按村民等级切换候选项，从插件内置的原版可能出现交易中选择当前职业的交易；点击“打开交易”会生成一个临时商人界面供玩家交易。当前初版交易目录按职业和等级提供保守白名单，后续可继续扩展到更完整的原版交易池。

配方：

```text
任意木板       任意木板     任意木板
任意木板       村民刷怪蛋   空气
任意木板       矿车         任意木栅栏
```

玩家获取过村民刷怪蛋后，会自动解锁简易村民繁殖机、简易村民交易站和僵尸村民刷怪蛋的配方书配方。

## 亡灵粉尘

“亡灵粉尘”是以火药为基底的自定义物品。僵尸和僵尸村民被玩家击杀时有 20% 基础概率掉落，每级抢夺额外提升 5% 概率；僵尸村民额外提升 20% 概率，小僵尸额外提升 20% 概率。亡灵粉尘带有插件标记，不能作为普通火药参与原版合成。

玩家获取过亡灵粉尘后，会自动解锁亡灵核心的配方书配方；如果玩家上线时已拥有亡灵粉尘或此前已解锁过，也会补发该配方解锁。

## 亡灵核心

“亡灵核心”是以烟火之星为基底的自定义物品。它带有插件标记，不能作为普通烟火之星参与原版合成。

配方：

```text
腐肉     腐肉     腐肉
腐肉     亡灵粉尘 腐肉
腐肉     腐肉     腐肉
```

右键使用亡灵核心会立刻消耗 1 个物品，并以使用地点为中心开启“亡灵潮”事件。事件期间：

1. 附近玩家会看到类似袭击事件的亡灵潮进度条，击杀 25 只事件僵尸后事件完成。
2. 事件中心会持续向外扩散更大范围的血红色粒子。
3. 每隔随机 20-100 tick，若事件僵尸未达到 5 只同时存在上限，会在事件中心周围 10-30 格的环带内直接刷新僵尸。生成时先确定 X/Z，再在与事件中心高度相差不超过 20 格的范围内寻找高度差最小的可落足位置。
4. 若连续 3 次找不到可生成僵尸的位置，事件中心会产生爆炸声和爆炸粒子，并立刻判定事件失败。
5. 事件僵尸不受自然刷怪时间、地点和刷怪上限影响，不会在日照下燃烧，并带有永久发光效果。它们会更偏向停留在事件中心附近，过度远离时会尝试返回中心。
6. 事件僵尸必定装备随机头盔，并持有随机武器。头盔从皮革、锁链、铁、金、铜中选取；武器从木、石、铁、金材质的剑、矛、斧中选取。由于原版没有分材质长矛，矛使用对应材质的铲作为外观代理。
7. 事件僵尸有 75% 概率按原版僵尸组长机制获得更高生命上限和生成增援属性，受攻击时可由原版机制概率召唤僵尸。
8. 事件僵尸若试图跨维度传送，会被直接移除；该移除不计入击杀、经验、掉落或事件进度。
9. 事件僵尸死亡时不会掉落任何物品，经验值为原先的 5 倍，并累计 1 点事件进度。
10. 成功完成后会播放胜利音效，进度条额外显示一段时间“胜利”。胜利展示期间血红色扩散粒子会继续渲染，事件中心会额外生成绿色倒计时粒子；随后隐藏进度条，并为所有参与击杀过事件僵尸的在线玩家各发放 1 个村民刷怪蛋。
11. 如果事件开启 1 小时后仍未完成，事件会自动失败。失败后不再刷新僵尸，残留的事件僵尸会在离开所有玩家加载范围后被删除。
12. 如果事件中心处于领地内，玩家必须拥有该领地的“开启事件”权限才可使用亡灵核心；权限不足时不会消耗亡灵核心。创造模式玩家使用亡灵核心开启亡灵潮时不会消耗物品。

## 权限

```text
xicesimpleindustry.admin
```

该权限默认仅 OP 拥有，用于重载插件配置。`/xicecustomitem give` 不直接依赖 Bukkit 权限节点，而是使用运行时配置中的 `access` 授权，便于 Web 权限页管理：

```yaml
access:
  default-allowed-actions: []
  players:
    <uuid>:
      name: ExamplePlayer
      actions:
        - give
```

## 部署

源码目录：

```text
plugins/XiceSimpleIndustry
```

Maven 构建产物：

```text
/opt/xicemc/runtime/plugins/XiceSimpleIndustry.jar
```

运行时配置：

```text
/opt/xicemc/runtime/plugins/XiceSimpleIndustry/config.yml
```

该插件依赖 `XiceCustomItem` 注册简易刷石机、简易村民繁殖机、简易村民交易站、亡灵粉尘和亡灵核心，依赖 `XiceHUD` 托管亡灵潮 BossBar，软依赖 `XiceClaim` 检查领地内“开启事件”权限。
