# XiceClaim

`XiceClaim` 是 XiceMCServer 的自制 Paper 领地保护插件，用于在玩家建设前提供基础安全感。当前版本以“清楚划定空间、阻止未授权操作、便于熟人协作”为目标，不做复杂领地经济。

## 当前功能

1. 玩家使用两个斜角点创建立方体领地。
2. `/claim pos1` 和 `/claim pos2` 可直接使用玩家当前位置，也可在命令后填写 `X Y Z` 指定坐标。
3. 当两个角点都已设置时，插件会在游戏内用仅自己可见的粒子预览框选空间，绿色粒子勾勒棱线，红色粒子显示面网格，闪烁约 `5` 秒。
4. 领地所有者可以授权或取消授权成员。
5. 非所有者、非成员不能在领地内破坏方块、放置方块、交互方块、打开容器、交互实体或使用桶装液体。
6. 领地默认禁止火焰蔓延、燃烧破坏、爆炸破坏，以及活塞运动跨越领地边界。
7. 管理员权限 `xiceclaim.admin` 可绕过领地保护并执行 `/claim reload`。
8. 领地插件提供以燧石为基底的“领地戒指”，只能通过 `/claim give ring` 获取，不添加原版合成配方。

## 玩家命令

```text
/claim pos1 [X Y Z]
/claim pos2 [X Y Z]
/claim create <名称>
/claim info [名称]
/claim list [玩家ID]
/claim trust <玩家>
/claim untrust <玩家>
/claim delete [名称]
/claim give ring
```

使用方式：

1. 站在建筑区域的一个角落，执行 `/claim pos1`，或执行 `/claim pos1 100 64 100` 指定第一个角点。
2. 站到对角，执行 `/claim pos2`，或执行 `/claim pos2 120 80 120` 指定第二个角点。
3. 两个角点都设置后，聊天栏会显示框选大小和是否合规，游戏内会显示粒子预览。
4. 执行 `/claim create home` 创建领地。
5. 站在领地内执行 `/claim trust 玩家名` 授权朋友共同建设。
6. 执行 `/claim info` 查询当前位置领地，也可以执行 `/claim info home` 查询指定领地。
7. 执行 `/claim list` 查询自己的所有领地；执行 `/claim list 玩家ID` 查询指定玩家的领地列表。

`/claim delete` 不填写名称时会尝试删除当前位置领地；如果当前位置不是自己的领地，会提示没有权限。填写名称时只能删除自己的同名领地。

`/claim give ring` 可获取一个以燧石为基底的“领地戒指”，需要在 `XiceClaim` 配置中为玩家分配 `give` 动作权限。空戒指右键打开领地创建 GUI，关闭 GUI 时会把坐标草稿保存回戒指；创建成功后戒指会发光并绑定到对应领地，右键可打开领地管理 GUI。戒指不能参与原版合成，避免被用于合成打火石或其它原版物品。

领地名称只能包含文字、数字、下划线和短横线，长度 `1-24`。领地戒指允许通过铁砧改名，戒指名称就是领地名称；当前领地名称全服唯一，创建时如果重名会失败。

## 默认限制

```yaml
limits:
  max-claims-per-player: 3
  min-size: 3
  max-horizontal-size: 128
  world-border: 50000
  world-min-y: -64
  world-max-y: 320
```

合规规则：

1. 长、宽、高都不得小于 `3` 格。
2. 长和宽不得大于 `128` 格。
3. X/Z 坐标不得超出 `-50000` 到 `50000`。
4. Y 坐标不得低于 `-64` 或高于 `320`。
5. 不能与任何已有领地重合。
6. 每个普通玩家最多拥有 `3` 个领地。

## 数据存储

领地数据存储在插件运行时目录：

```text
/opt/xicemc/runtime/plugins/XiceClaim/claims.yml
```

该文件属于运行时数据，不提交到 Git。日常备份脚本会随插件运行时配置一起备份。

## 当前限制

1. 暂不提供 Web 可视化地图。
2. 暂不提供领地转让、子领地、租售和经济系统。
3. 暂不提供告示牌或工具选区，当前通过命令设置角点。
4. 粒子预览只对执行命令的玩家可见，主要用于确认边界，不作为长期显示。
5. 爆炸保护会移除爆炸列表中的领地方块，不区分爆炸来源。
