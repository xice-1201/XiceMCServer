# XiceClaim

`XiceClaim` 是 XiceMCServer 的自制 Paper 领地保护插件，用于在玩家建设前提供基础安全感。当前版本以“清楚划定空间、阻止未授权操作、便于熟人协作”为目标，不做复杂领地经济。

## 当前功能

1. 玩家使用“领地戒指”GUI 选择两个斜角点并创建立方体领地。
2. 空领地戒指右键打开创建 GUI，绑定后的领地戒指右键打开对应领地管理 GUI。玩家进服时会自动解锁领地戒指配方，可直接在配方书中查看。
3. GUI 预览会在游戏内用仅自己可见的粒子预览框选空间，绿色粒子勾勒棱线，红色粒子显示面网格，闪烁约 `5` 秒。
4. 领地所有者可以授权或取消授权成员。
5. 领地权限支持“禁止未授权”“允许所有人”“全体禁止”三种状态；只有“允许所有人”会显示附魔光效。
6. 非所有者、非成员默认不能在领地内放置方块、破坏方块、交互方块、打开容器、交互实体、使用水桶或岩浆桶，也默认不能伤害领地内动物或怪物。
7. 领地默认禁止未授权火焰蔓延、燃烧破坏、爆炸破坏，以及活塞运动跨越领地边界；领地内使用活塞、刷新动物和刷新怪物默认允许所有人。领地所有者可在领地戒指的“管理领地功能”菜单中逐项切换这些行为。
8. 管理员权限 `xiceclaim.admin` 可管理所有领地并执行 `/claim reload`；“全体禁止”状态仍会拦截实际行为。
9. 领地插件提供以燧石为基底的“领地戒指”，可通过 `/claim give ring` 获取，也可用铁锭、钻石、铁粒和线合成初始未绑定戒指。
10. 玩家进入任一领地时，会在对话栏收到领地名称和所有者提示，并自动展示该领地范围；离开领地时只显示文字提示。
11. 服务器会通过资源包下发领地戒指的专用物品图标，资源包入口为 `/resourcepacks/xiceclaim.zip`。
12. 插件提供以信标为基底的“领地图腾”，可通过 `/claim give totem` 获取，也可在获得过领地戒指并解锁配方后合成。领地图腾放置后生成上下两个带插件标记的方块实体作为占位，并使用展示实体渲染临时外观。

## 玩家命令

```text
/claim give ring
/claim give totem
```

使用方式：

1. 通过 `/claim give ring` 获取领地戒指。
2. 使用铁砧为领地戒指改名，戒指名称就是要创建的领地名称。
3. 手持空戒指右键打开创建 GUI，通过 `-50`、`-10`、`-1`、`+1`、`+10`、`+50` 按钮微调两个角点坐标，或使用当前位置写入角点。
4. GUI 中间告示牌会显示框选大小、体积、合规状态和创建失败原因。
5. 点击预览会保存当前草稿、关闭 GUI，并显示粒子边界。
6. 创建成功后戒指会发光并绑定领地，右键可进入管理 GUI。
7. 绑定戒指的管理 GUI 先展示主菜单，可进入授权玩家、领地功能、范围预览和删除确认等子菜单。“管理领地功能”中发光的项目表示允许所有人，无光项目表示禁止未授权或全体禁止，点击可循环切换。该菜单只展示权限项，不包含范围预览和删除领地按钮。删除领地需要二次确认，删除后戒指恢复为空戒指但保留当前名称。
8. 未绑定领地的领地戒指可通过“绑定至领地”按钮绑定到自己拥有或被授权的领地。

领地图腾当前只实现基础放置与回收，不接入领地范围、权限、增益或其它玩法功能：

1. 手持领地图腾右键放置时，会占用放置目标处 `1 x 1 x 2` 空间；放置目标下方必须有方块支撑。
2. 两格空间中原版正常放置方块时可替换的内容，例如空气、草和花，也会被领地图腾替换。
3. 领地图腾上下两格都使用带插件持久标记的方块实体保存逻辑身份，并通过展示实体渲染临时信标外观；普通信标的合成、放置、交互和破坏不受影响。
4. 右键已放置的领地图腾不会打开信标界面。
5. 破坏任意一格都会移除整个领地图腾；生存模式掉落一个领地图腾，创造模式不掉落。
6. 领地图腾本体不会被爆炸炸毁；如果领地图腾依附的下方方块被破坏或消失，领地图腾会像火把一样整体掉落为一个物品。
7. 领地图腾的物品形态显式关闭附魔光效；放置后的外观由展示实体控制，不再依赖真实信标方块发光。

合成配方：

领地戒指：

```text
铁锭 钻石 铁锭
铁粒  空  铁粒
 空   线   空
```

领地图腾：

```text
下界石英 下界之星 下界石英
金锭     末影珍珠 金锭
石英台阶 石英台阶 石英台阶
```

领地图腾配方默认不会在工作台配方书中显示。玩家获得过领地戒指后，插件会解锁领地图腾配方。

旧的 `/claim pos1`、`/claim pos2`、`/claim create`、`/claim info`、`/claim list`、`/claim trust`、`/claim untrust`、`/claim delete` 玩家指令已关闭，领地创建与管理完全使用领地戒指 GUI。

`/claim give ring` 可获取一个以燧石为基底的“领地戒指”，需要在 `XiceClaim` 配置中为玩家分配 `give` 动作权限；玩家也可以通过上方配方合成初始未绑定戒指。空戒指右键打开领地创建 GUI，关闭 GUI 时会把坐标草稿保存回戒指；创建成功后戒指会发光并绑定到对应领地，右键可打开领地管理 GUI。领地戒指不能作为原料继续参与原版合成，避免被用于合成打火石或其它原版物品。

领地名称只能包含文字、数字、下划线和短横线，长度 `1-24`。领地戒指允许通过铁砧改名，戒指名称就是领地名称；当前领地名称全服唯一，创建时如果重名会失败。

## 默认限制

```yaml
limits:
  max-claims-per-player: 3
  min-size: 3
  max-horizontal-size: 128
  world-border: 50000
  world-min-y: -64
  world-max-y: 320
```

合规规则：

1. 长、宽、高都不得小于 `3` 格。
2. 长和宽不得大于 `128` 格。
3. X/Z 坐标不得超出 `-50000` 到 `50000`。
4. Y 坐标不得低于 `-64` 或高于 `320`。
5. 不能与任何已有领地重合。
6. 每个普通玩家最多拥有 `3` 个领地。

## 数据存储

领地数据存储在插件运行时目录：

```text
/opt/xicemc/runtime/plugins/XiceClaim/claims.yml
```

该文件属于运行时数据，不提交到 Git。日常备份脚本会随插件运行时配置一起备份。

## 当前限制

1. 暂不提供 Web 可视化地图。
2. 暂不提供领地转让、子领地、租售和经济系统。
3. 粒子预览只对执行操作的玩家可见，主要用于确认边界，不作为长期显示。
4. 爆炸保护会移除爆炸列表中的领地方块，不区分爆炸来源。
