# 世界规划 v0.2

## 阶段目标

当前阶段的目标是先开起来一个稳定、干净、低维护成本的 Java 版原版生存服务器。

本阶段暂不加入 RPG 内容、资源世界、PVP 专用世界、副本世界、测试世界或复杂经济系统。服务器先以熟人白名单为主，核心体验围绕原版生存、长期建筑、探索、合作和基础公共设施展开。

后续如果服务器稳定运行、玩家活跃度足够，再单独讨论 RPG、资源世界、活动世界和公开开放方案。

## 世界列表

| 世界 | 持久性 | 主要用途 | PVP | 重置策略 |
| --- | --- | --- | --- | --- |
| `main` | 永久 | 主世界生存、家园、建筑、城镇、公共设施 | 默认关闭 | 不重置 |
| `main_nether` | 永久 | 下界探索、下界交通、烈焰人农场、下界资源 | 默认关闭 | 原则上不重置 |
| `main_the_end` | 永久 | 龙战、末地探索、末地建筑、鞘翅和潜影壳获取 | 默认关闭 | 原则上不重置 |

## 主世界定位

`main` 是服务器的核心世界，也是玩家长期资产所在的世界。

主世界应承担：

1. 玩家出生点和基础公共设施。
2. 玩家家园、村落、城镇和长期建筑。
3. 农场、道路、铁路、传送点等公共工程。
4. 普通生存探索和原版进度推进。

主世界不做周期性重置。除非出现严重损坏、存档不可恢复或所有玩家共同同意，否则不考虑重置。

## 下界定位

`main_nether` 是长期下界世界，主要用于探索、交通和原版资源获取。

下界应承担：

1. 下界交通网络。
2. 烈焰人、猪灵、凋灵骷髅等原版内容。
3. 石英、荧石、远古残骸等下界资源采集。
4. 与主世界长期基地配套的下界设施。

下界原则上不重置。由于没有资源世界，玩家应尽量避免在公共交通范围内进行大面积破坏性采集。

## 末地定位

`main_the_end` 是长期末地世界，主要用于龙战、末地探索和末地资源获取。

末地应承担：

1. 末影龙挑战。
2. 末地城探索。
3. 鞘翅和潜影壳获取。
4. 末地公共传送、桥梁或安全通道建设。
5. 未来可能的末地建筑项目。

末地原则上不重置。如果鞘翅、潜影壳等资源明显不足，优先考虑扩大探索范围、制定分配规则或后续新增资源末地，而不是直接重置永久末地。

## PVP 政策

当前阶段默认关闭 PVP。

允许 PVP 的情况暂不开放自动化规则，仅保留未来可能性：

1. 双方明确同意的临时切磋。
2. 未来单独规划的活动或竞技场。

未经允许击杀、陷阱击杀、抢夺死亡掉落、复活点蹲守等行为视为违规。

## 建筑与资源原则

由于当前阶段没有资源世界，所有世界都应被视为长期世界。

资源采集建议：

1. 大规模挖矿尽量远离玩家聚居区和公共景观。
2. 大量采集沙子、泥土、树木等地表资源时，尽量修复地形或选择远离公共区域的位置。
3. 不要破坏公共道路、公共农场、出生点、传送设施和其他玩家建筑。
4. 大型农场、红石机器和高频装置应提前沟通，避免造成服务器卡顿。

## 保护策略

第一阶段保护策略应简单但可靠：

1. 使用白名单和正版验证。
2. 尽量不给玩家 OP。
3. 使用 LuckPerms 管理具体权限。
4. 使用 CoreProtect 记录方块、容器和玩家行为，方便查证与回滚。
5. 如果出现公共区域或出生点建设需求，再使用 WorldGuard 保护关键区域。
6. 暂不急于加入领地插件，等玩家数量和建筑规模上来后再评估。

## 备份策略

三个世界都是长期世界，因此备份优先级很高。

建议策略：

1. 每日自动备份 `main`、`main_nether`、`main_the_end`。
2. 每周保留一份完整备份。
3. 每次更新服务端核心、插件或关键配置前额外备份。
4. 备份至少保留在服务器外部的一个位置。
5. 每月至少测试一次恢复流程。

## 当前阶段不做的内容

以下内容暂不纳入第一阶段：

1. RPG 世界。
2. 资源世界。
3. PVP 专用世界。
4. 副本、任务、NPC 和自定义 Boss。
5. 复杂经济系统。
6. 公开服宣传和大规模玩家管理。

## 下一阶段触发条件

只有当基础原版服务器稳定运行一段时间后，再考虑扩展。

可考虑进入下一阶段的条件：

1. 服务器稳定运行至少两到四周。
2. 备份、重启、权限和日志审计流程已经验证。
3. 玩家明确提出资源世界、RPG 或活动需求。
4. 当前世界没有明显管理混乱或性能问题。
5. 管理者每周维护一次仍然足够。
