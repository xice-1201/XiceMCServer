# Git 提交规范

## 提交标题

提交标题必须使用中文，并使用类型前缀标明变更类型。

格式：

```text
TYPE: 中文提交标题
```

示例：

```text
FEAT: 建立版本号与提交规范
FIX: 修正世界规划中的重置说明
DOCS: 更新服务器运维说明
CHORE: 调整仓库忽略规则
```

## 常用类型

| 类型 | 用途 |
| --- | --- |
| `FEAT` | 新增功能、流程、配置能力或重要文档结构 |
| `FIX` | 修复错误、遗漏、冲突或不正确的配置 |
| `DOCS` | 仅修改文档内容 |
| `CHORE` | 仓库维护、格式、忽略规则、脚手架等杂项 |
| `CONFIG` | 修改服务器、插件或部署配置 |
| `OPS` | 修改运维、备份、部署、重启相关内容 |
| `SECURITY` | 修改权限、安全、密钥处理或访问控制 |

## 提交内容

提交正文应使用中文，说明：

1. 本次变更做了什么。
2. 为什么需要这次变更。
3. 是否影响部署、备份、权限或世界数据。

较小变更可以只有标题；涉及版本、部署、备份、安全或世界规划时，应写提交正文。

## 历史提交

已经推送到远程仓库的历史提交不主动改写。新的提交从本规范建立后开始执行。
