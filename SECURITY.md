# 安全政策

## 支持范围

本仓库当前只维护 `main` 分支。安全修复会优先合入 `main`，并通过 GitHub Actions、CodeQL、Dependabot 和 secret scanning 进行检查。

## 报告安全问题

如果你发现安全漏洞、密钥泄露、隐私数据泄露、权限绕过、远程执行、数据库访问风险或服务器部署风险，请不要在公开 issue 中贴出可直接利用的细节、真实密钥、玩家隐私数据或服务器敏感信息。

请通过 GitHub 私下联系仓库维护者，或先提交一个不包含利用细节的 issue，说明需要私下沟通安全问题。

## 处理原则

维护者会优先处理以下问题：

1. 可能导致服务器控制权泄露的问题。
2. 可能导致密钥、数据库、玩家隐私数据泄露的问题。
3. 可能绕过 Web 后台、白名单、领地保护或指令权限的问题。
4. GitHub Actions、部署脚本、依赖或配置中的高危漏洞。

安全修复合入前应尽量通过 pull request、必过检查和签名提交规则。紧急情况下可先完成临时缓解，再补充正式修复。

## 不公开的信息

以下内容不应出现在 issue、pull request、commit、日志截图或讨论中：

1. SSH 私钥、API 密钥、数据库密码、RCON 密钥、Cookie、Token。
2. 真实服务器运行数据、数据库导出、备份包。
3. 玩家 UUID、IP、联系方式、聊天记录等隐私数据，除非已脱敏。
4. 尚未修复漏洞的完整利用步骤。
