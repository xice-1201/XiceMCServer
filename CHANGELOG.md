# 更新记录

## 0.0.8

类型：个人测试版本，尚未正式上线的第八个更新。

### 新增

1. Web 登录后侧边栏新增“举报”页面入口，页面暂留空。
2. Web 登录后侧边栏新增“服务器状态”页面，展示开服状态、在线玩家、磁盘空间、内存占用、备份文件列表和最近日志 ERROR 情况。
3. 新增审计日志清理脚本，默认删除超过 `3` 天的审计记录。

### 调整

1. 每日备份与更新维护时间从 `00:00` 调整为 `16:00`。
2. 每日维护流程在清理备份后同步清理过期审计日志。
3. Web 审计查询时间选择限制在最近 `3` 天内，玩家可在该范围内自由选择更细粒度时间段。

## 0.0.7

类型：个人测试版本，尚未正式上线的第七个更新。

### 新增

1. Web 页面预留 Microsoft 身份验证框架，默认关闭。
2. 新增 `/auth/microsoft/start` 和 `/auth/microsoft/callback`，用于 Microsoft OAuth 登录回调。
3. Microsoft 登录流程预留 PKCE state 校验，并继续换取 Xbox Live、XSTS 和 Minecraft Java Profile。
4. Microsoft 注册白名单流程保留邀请码限制，通过正版 Java Profile 得到玩家名和 UUID 后再加入白名单。

### 说明

该能力需要域名、HTTPS、Microsoft 应用注册和服务器私密环境变量配置完成后再启用。

## 0.0.6

类型：个人测试版本，尚未正式上线的第六个更新。

### 新增

1. 将 Web 首页重构为登录页，白名单内玩家可使用游戏 ID 登录。
2. 白名单注册功能移动到 `/register`。
3. 登录后首页展示玩家 UUID、注册时间、上次登出地点和累计游玩时间。
4. Web 侧新增“操作查询”页面，查询 `XiceAuditLog` 审计记录。
5. 审计插件新增玩家进入和退出记录，为 Web 首页提供会话数据。

### 优化

1. 操作查询默认不展示数据，必须设置筛选条件。
2. 查询采用参数化 SQL、固定最大页大小和游标分页，不执行总数统计。
3. 为审计查询补充玩家、动作、目标、物品和位置相关索引。

## 0.0.5

类型：个人测试版本，尚未正式上线的第五个更新。

### 新增

1. 新增 `XiceAuditLog` 审计插件，记录玩家方块破坏、方块放置、容器放入和容器取出。
2. 审计数据直接写入 PostgreSQL，不使用 SQLite。
3. 审计写入采用异步队列、单独写入线程和批量提交，避免在 Minecraft 主线程执行数据库 I/O。

### 说明

当前版本只记录审计数据，不提供游戏内查询和回滚。

## 0.0.4

类型：个人测试版本，尚未正式上线的第四个更新。

### 新增

1. `XiceTextArranger` 新增正版验证失败提示配置。
2. 当 Paper 能捕获到认证失败拒绝消息，并且原始拒绝文本匹配配置时，会提示玩家使用正版启动器登录 Microsoft 账号后重试。

### 说明

部分正版验证失败可能发生在 Paper 插件事件之前，此时插件无法改写提示，但 `online-mode=true` 仍会拒绝未通过正版验证的连接。

## 0.0.3

类型：个人测试版本，尚未正式上线的第三个更新。

### 新增

1. 新增 `XiceTextArranger` 插件，用于添加、重写或删除部分游戏系统消息。
2. 新增白名单拒绝提示配置，引导未加入白名单的玩家访问 `http://150.158.93.80/` 注册白名单。
3. 预留玩家加入和退出消息的 `keep`、`delete`、`rewrite`、`add` 配置能力。

## 0.0.2

类型：个人测试版本，尚未正式上线的第二个更新。

### 新增

1. `XiceCommandControl` 新增 `/xcc reload`，允许在游戏内重新加载插件配置。
2. `XiceCommandControl` 新增 `/xcc list [玩家名或UUID]`，允许查询玩家拥有的受控指令。

### 调整

1. 玩家专属权限改为以 UUID 登记，玩家名仅用于显示和查询。
2. `ExamplePlayer` 获得 `/creative`、`/xcc reload`、`/xcc list` 权限。
3. 部署脚本会同步自制插件的默认配置到服务器运行目录。

## 0.0.1

类型：个人测试版本，尚未正式上线的第一个更新。

### 新增

1. 确立服务器第一阶段目标：先启动基础 Java 版原版生存服务器。
2. 世界范围暂定为 `main`、`main_nether`、`main_the_end`。
3. 暂不加入 RPG、资源世界、PVP 专用世界、副本和复杂经济系统。
4. 建立版本号文件 `VERSION`。
5. 建立 Git 提交规范文档。

### 说明

版本号格式暂定为 `个人测试版本.上线阶段.更新序号`。

当前 `0.0.1` 表示：

1. 第一个 `0`：个人测试版本。
2. 第二个 `0`：尚未正式上线。
3. 第三个 `1`：第一个更新。
